## Key Components

### Frontend (`frontend/`)

- **Next.js App Router** — file-based routing with server and client components
- **shadcn/ui** — accessible, customizable component library built on Radix UI
- **Framer Motion** — animations for the office map and UI transitions
- **WebSocket client** — real-time bi-directional communication with the backend

### Backend (`backend/`)

- **FastAPI** — async Python web framework for REST and WebSocket endpoints
- **SQLAlchemy (async)** — ORM with aiosqlite for SQLite database access
- **LangGraph** — agent execution loop with tool calling and state management
- **LiteLLM** — unified interface to OpenAI, Anthropic, and other LLM providers
- **ChromaDB** — vector database for agent memory and RAG

## Data Flow

### CEO Message Routing

1. User sends a message via the WebSocket chat interface
2. The **Router** (powered by GPT-4o-mini) analyzes the message intent
3. The router selects the appropriate department and agent
4. The **Runner** executes the agent using LangGraph
5. The agent may call tools (RAG search, social posting, etc.)
6. Responses stream back to the frontend via WebSocket events

### Agent Memory

Each agent has two memory layers:

- **Personal memory** — a ChromaDB collection scoped to the agent, storing conversation history and uploaded documents
- **Company knowledge base** — a shared ChromaDB collection accessible to all agents via the `search_knowledge` tool

### LLM Configuration

| Purpose | Model | Provider |
|---------|-------|----------|
| Lead agents | GPT-4o | OpenAI |
| Creative agents | Claude Sonnet | Anthropic |
| Message routing | GPT-4o-mini | OpenAI |
| Agent generation | GPT-4o | OpenAI |
| Embeddings | text-embedding-3-small | OpenAI |

## Project Structure
